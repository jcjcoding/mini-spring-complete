四、demo 阶段核心技术实现要点
登录流程：
前端：小程序端调用 wx.login 获取 code，传递给后端；
后端：通过 weixin-java-miniapp 调用微信接口获取 openid，生成 JWT token 返回前端；
前端：存储 token，后续请求（如打卡、镇压心魔）携带 token 到后端验证。
核心业务逻辑：
灵气值 / 心魔值计算：后端封装业务层（Service），通过 MyBatis-Plus 操作 MySQL，Redis 缓存当前值（如用户最新灵气值），保证响应速度；
打卡记录：MySQL 设计 check_in_record 表，关联用户 ID，存储打卡类型（修炼 / 历练）、灵气值、打卡时间，前端日历组件读取该表数据展示印记；
境界升级：后端预设境界阈值（炼气 100、筑基 500 等），灵气值达到阈值时触发「突破」逻辑，返回提示给前端。
性能适配：
demo 阶段无需考虑高并发，Redis 仅缓存高频数据（如用户当前灵气值、连续打卡天数）；
小程序端接口请求超时设置为 5 秒，后端接口响应控制在 1 秒内（符合需求文档的性能要求）。
总结
核心技术栈：后端以 SpringBoot 2.7.x + MyBatis-Plus + Redis + WxJava 为核心，前端用微信小程序原生框架 + ColorUI，数据库选用 MySQL 8.0；
demo 轻量化原则：避免分布式、高并发等复杂架构，聚焦基础功能实现，优先保证开发效率和交互体验；
关键依赖：weixin-java-miniapp 是小程序与后端交互的核心，Redis 是满足性能需求的关键，ColorUI 快速实现修仙风视觉效果。
